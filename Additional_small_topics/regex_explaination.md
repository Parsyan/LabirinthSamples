## Regex (Regular Expressions)

#### Ի՞նչ, Որտե՞ղ, Ե՞րբ

Պատկերացրու ունես տեքստ, և քեզ պետք է ոչ թե ամբողջ տեքստը, այլ **մի կոնկրետ օրինաչափությամբ հատվածներ**։

Օրինակ՝  
- բոլոր թվերը  
- բոլոր email-ները  
- բոլոր բառերը, որ սկսվում են մեծատառով  

Եթե փորձես սա անել սովորական `String` մեթոդներով (`contains`, `indexOf`, `substring`)՝ արագ կհասկանաս, որ ամեն բան դառնում է խառն ու դժվար կառավարելի։

Այստեղ է հայտնվում **Regex-ը** — տեքստի մեջ pattern (օրինաչափություն) նկարագրելու լեզու։

---

### Ինչ է Regex-ը իրականում

Regex-ը **մինի լեզու է**, որով դու ասում ես.

> «գտիր ինձ այն տեքստը, որը համապատասխանում է այս կանոնին»

Օրինակ՝

- `\d` → ցանկացած թիվ  
- `\d+` → մեկ կամ ավելի թվեր  
- `[a-z]` → փոքրատառ լատիներեն տառ  
- `[A-Z]` → մեծատառ  
- `.` → ցանկացած սիմվոլ  

Այսինքն դու **չես փնտրում կոնկրետ արժեք**, այլ **նկարագրում ես ձևը**։

---

### Որտե՞ղ է օգտագործվում

Regex-ը օգտագործվում է այն բոլոր տեղերում, որտեղ կա **տեքստի մշակում**.

- Form validation (email, password)
- Log parsing (սերվերի log-ներ)
- Data cleaning (աղտոտ տվյալների մաքրում)
- Search համակարգեր
- Compiler-ներ, interpreters

Կարճ՝  
> որտեղ կա տեքստ → այնտեղ Regex-ը ունի տեղ

Բայց կա կարևոր nuance.

Regex-ը լավ է, երբ խնդիրը **pattern է**  
ոչ թե բարդ business logic

---

### Երբ օգտագործել (իրական իրավիճակ)

#### Խնդիր

Ունենք օգտատերերի input, և պետք է ստուգենք՝ արդյոք սա ճիշտ email է։

Եթե փորձենք առանց Regex՝

- պետք է ստուգենք `@` կա՞  
- հետո `.`
- հետո դիրքերը  
- հետո edge cases  

Կոդը արագ կդառնա messy։

---

#### Լուծում Regex-ով

```java
import java.util.regex.*;

public class Main {
    public static void main(String[] args) {
        String email = "test123@gmail.com";

        String pattern = "^[\\w.-]+@[\\w.-]+\\.[a-zA-Z]{2,}$";

        boolean isValid = Pattern.matches(pattern, email);

        System.out.println(isValid);
    }
}
